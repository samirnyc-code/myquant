How the options desk runs itself

A tour of Samir's automated SPX 0DTE paper-trading day · all times are US Central (exchange time) · July 2026

Every weekday, a Windows machine wakes a chain of ~20 scheduled jobs that log into Interactive Brokers, pull the day's gamma levels, write a trading gameplan, watch price all day, paper-trade the plan's triggers, and file a report card at the close. Nobody has to touch it. This page explains the pieces in the order they fire.

The big picture

Four ingredients, one loop:

MenthorQ levels are dealer-gamma price levels for SPX — walls where options market-makers' hedging tends to slow or pin price (Call Resistance, Put Support, the Hedge Volatility Line, the gamma wall). The whole desk is one experiment: do these levels plus a pre-written plan make money, traded mechanically on paper?

Two kinds of jobs

Everything is a Windows Task Scheduler job, but they come in two modes:

One-shots run, write a file, and exit — like the gameplan writer. If one shows "failed" in Task Scheduler it usually just means "attention", not "crashed": the health check and EOD report deliberately exit non-zero when the desk isn't fully green.

Daemons start in the morning and stay alive until the close — the spot feed, the trigger daemon, the position marker. They're the desk's eyes and hands during market hours.

The day, hour by hour

  • 07:30
    MQ Mine — archives the full MenthorQ surface (levels, per-strike gamma, vol) for ~25 tickers into data/menthorq/mine/.Research library, not used for today's trading — it's how the level studies get their history.
  • 08:00
    Gateway login — IBC auto-starts IB Gateway and logs into the paper account, exposing the API on port 4002.Guarded: if the gateway is already up it exits without touching the session.
  • 08:15–08:17
    Level bookkeeping — scrapes level hold-rate stats and backfills recent level history.
  • 08:20
    Gateway ensure — belt-and-braces: probes port 4002 and relaunches the gateway only if it's down.Lesson from a July incident: "logged in" and "API port up" are not the same thing.
  • 08:25
    Dashboard daemon — the live desk page on :8600. Backstop start; if Mission Control already runs it, this exits clean.
  • 08:26
    Spot feed daemon — streams live SPX/ES/VIX into live.json until ~16:20. Marks watch daemon — re-prices every open position every 2 minutes (P&L, prob. of profit, expected value).
  • 08:27
    Levels fetch — today's MenthorQ levels for SPX → mq_levels_today.json.
  • 08:28
    Gameplan — the brain. Reads levels + spot, classifies the gamma regime, writes 4 price paths (A–D) and ~7 armed triggers, each with a grade and a fire condition. Sim daemon daemon — separately runs the one validated strategy (STMR bull put spread) and makes its single go/no-go decision at 14:59.
  • 08:32–08:33
    Levels engine + Trigger daemon daemon — the hands. Watches the tape against the gameplan's conditions ("PS0 touched from above", "broke below HVL") and executes the paper trade the moment one fires. Runs until 15:00.
  • 08:35–08:40
    Gamma scanner — same regime read across 14 tickers. Health check — 8-point green/red status; sends a notification if anything's down.
  • 14:59
    STMR decision — the sim daemon computes its end-of-day signal (K8 < 15 and spot > SMA100) and sells a 14-DTE put spread if it fires. Logged to decisions.csv.
  • 15:15–15:20
    Postmortem — plan vs. actual: which scenario played out, which triggers fired, contradictions flagged. EOD report — HTML report card of the whole day.
  • 15:45–16:15
    Evening harvest — MQ harvest, QUIN harvest, levels database update. Archival, feeds the research side.

What "a trade" means here

Everything is paper — an IB paper account plus an internal fill-model log. A trigger like "PS0 first-touch fade" sells a defined-risk SPX put credit spread when price first touches Put Support from above. Entries, quotes during the fill window, and exits all get logged so fill realism can be audited later (a hard-learned house rule: high win rates are treated as bugs until proven on real quotes).

One strategy is validated; the rest are experiments. Only the 14:59 STMR bull put spread has survived a proper backtest. The gameplan's day-triggers (fades, flies, straddles) are hypothesis-testing with grades attached — the point is the postmortem, not the P&L.

How it heals itself

FailureWhat catches it
Gateway down / API port deadgateway_ensure at 08:20, plus every IB script auto-relaunches IBC and retries once before giving up.
Dashboard already runningThe scheduled start probes port 8600 first and exits clean — no duplicate servers.
Second gateway login kicking the firstLogin script checks port 4002 before launching; already-up = do nothing.
Anything else red08:40 health check + EOD report both flag it; notifications land in notifications.log.

Where to look

Live desk dashboard — port 8600 (shared over Tailscale; Samir has the keyed link). Positions, live P&L, the day's gameplan, LIVE pill when the feed is fresh.
Mission Control — port 8590, start/stop buttons for all dashboards.
Files — everything lands in data/options_sim/: gameplan_YYYYMMDD.json, scanner_YYYYMMDD.json, eod_report_YYYYMMDD.html, decisions.csv.
